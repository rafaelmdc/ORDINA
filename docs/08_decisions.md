# 08 — Resolved decisions

> This doc records decisions taken on the open questions in [`06_open_questions.md`](06_open_questions.md), from a mix of reasoning and the user's domain/priority calls. Where an item here conflicts with an earlier doc, **this doc wins** and the earlier doc has been updated to match. Items still genuinely open are listed at the end.

## A. Scientific framing

- **A1 — First goal: recover-and-explain-known (validation-first).** *(user call)* The first iteration's headline claim is: the multiplex **re-derives known disease–organism associations** and annotates each with the **layers that explain it** (shared metabolism, niche, observed co-occurrence…). Rationale: this is the credibility gate — novel predictions are only trustworthy once recovery provably beats the null. Discovery (the layer-discordance heuristic) is explicitly the *second* iteration.
- **A2 — "Decent enough" exit criterion.** *(resolves Q12)* The docs phase ends and building begins when: (a) a **recovery metric** — known-association recovery beating the phylogeny null on a held-out split — is defined and measurable (**now specified in full in [`09_recovery_metric.md`](09_recovery_metric.md)**; the bar is beating the phylogeny, study-effort, *and* enrichment-coverage baselines); (b) the layer set, operating rank, and disease vocabulary are fixed (this doc); and (c) the coverage audit (D2) confirms enough enriched nodes to be worth it (**done — directional: ~87% of species carry ≥1 enrichment layer, with a confirmed study-effort bias that the recovery metric now explicitly controls for**). The milestone is tied to A1, not to feature count.
- **A3 — Operational "novel."** *(resolves Q5)* A candidate is *novel* if it is **absent from every curated seed source at snapshot time** and is surfaced purely from enrichment layers **with the disease layer held out**. Deferred to iteration 2 per A1, but defined now so the firewall is unambiguous.
- **A4 — The disease data can be used many ways; the only hard rule is "don't grade your own homework."** *(user call, refines A3)* An earlier draft said "disease is always the target, enrichment is always the features, never mix." That was too strict. The disease data is not permanently a "target" or a "feature" — its role changes depending on which question we are asking, and **we can produce several separate findings from the same data.** Examples of legitimate, different findings:
  1. *Does biology explain a known disease link?* — Here the disease link we are testing is the answer; we hide it and try to predict it from metabolism/traits.
  2. *Does the disease network have its own shape?* — Here we study the disease layer directly (which organisms group together because they show up in the same diseases).
  3. *Does predicted interaction match observed interaction?* — Here disease is not involved at all (metabolism vs. co-occurrence).
  4. *Can one disease's pattern help predict a different disease?* — Here disease is used as both an input and an output, but for **different** diseases.
  5. *Filling gaps for "dark" organisms* — an organism with no biology data of its own can still get a **guess** based on the organisms it shares diseases with (see B5).

  **The one thing you may never do:** inside a *single* test, use the exact thing you are trying to predict as one of the inputs you predict it from. (Plainly: if you are trying to predict "Crohn's," nothing you built out of Crohn's data is allowed to help make that prediction — otherwise the test always "passes" and means nothing. This is called a data leak.)

  **Two ways a leak sneaks in here, so guard against both:**
  - **Similar diseases.** Crohn's and ulcerative colitis share a lot of biology, so "I predicted UC using Crohn's data — different disease, so it's fine" can still be cheating. When we split data into a part we learn from and a part we test on, that split must be drawn carefully so near-identical diseases don't end up on both sides.
  - **Remember where a guess came from.** If we fill in a missing trait for a "dark" organism using its disease neighbours (finding #5), we must **label that trait as "guessed from disease data."** Then it is not allowed to be used later in any test where that same disease is the thing being predicted.

  So the rule is not "never use disease as an input." It is: **keep track of where every piece of data came from, and make sure a test never secretly sees its own answer.** Everything else is allowed, and gives us several findings instead of one. This directly shapes how the recovery metric's train/test split is drawn (item 2).

## B. Modelling

- **B1 — Operating rank: dual-rank.** *(user call, Q3)* Nodes are maintained at **both genus and species**. **Primary analysis runs at genus** (most robust, best coverage — 16S disease data is genus-reliable); **drill-down to species** where the data supports it. Consequences, now mandatory:
  - Every **layer factory must declare its native rank and an explicit aggregation rule to genus** (e.g. metabolic reconstructions are genome/strain-level and must define how strain→species→genus aggregation works).
  - Rollup uses `TaxonClosure`, and must **preserve and surface species-level contradictions** rather than averaging them away (a genus can look "neutral" while its species disagree — that disagreement is a finding, not noise).
  - Default reporting is at genus; species detail is a drill-down, not a separate network.
- **B2 — Cross-layer weight normalization (made concrete).** *(resolves Q2)* Each layer produces weights in its own units — a metabolic complementarity score, a trait Jaccard, and a phylogenetic proximity are not the same currency — so before any layer is combined with another they must be put on a common scale. The concrete rules:
  - **Default method: rank-based normalization to [0,1], within each layer.** Take a layer's edge weights, sort them, and replace each weight by its rank position scaled to the 0–1 range (ties share the average rank). Why rank and not the obvious alternatives:
    - **min–max scaling** is fragile — one outlier weight squashes everything else toward zero;
    - **z-score** produces negative numbers, which break the random walk (it needs non-negative edge weights) and assumes a symmetric spread the data won't have;
    - **rank** is immune to outliers and to whatever odd shape a layer's weight distribution has, and it keeps everything non-negative. Its one cost — it throws away the exact magnitudes, keeping only the order — is acceptable for iteration 1 and is precisely what makes two very differently-scaled layers comparable. (A stronger option, quantile-matching to a common reference distribution, is held in reserve if plain rank proves too coarse.)
  - **Coverage-aware:** the ranking is computed over a layer's **measured edges only**. A `measured_absent` pair is 0; an `unmeasured` pair is masked, never 0 (B3/B5).
  - **Signed layers:** normalize the **magnitude** to [0,1] and keep the **sign** in its own column (the schema's `sign` field, [`03`](03_layer_factory_and_catalog.md) §1a) — so a strong co-exclusion and a strong co-occurrence get the same magnitude but opposite sign, and neither is confused with a weak edge.
  - **Applied at build time** and recorded in the layer manifest (`normalization`); **raw weights are always retained as provenance.**
  - **Inter-layer coupling** (how strongly a node in one layer connects to itself in another, for the combined supra-network) is a **swept hyperparameter, not a hidden constant**: try a range of values, report how the results move across that range, and pick by downstream stability — never assume one value. Layers are weighted **equally by default**; any supervised per-layer weighting is trained on the training split only (never the held-out data), to avoid the leak in A4.
- **B3 — Measured-absent vs unmeasured.** *(resolves Q4)* Three-state edge presence (present / measured-absent / unmeasured) via the layer coverage flag. Analyses run on the **measured-induced subgraph** per layer; unmeasured is never silently treated as zero-similarity.
- **B4 — Multiplex vs heterogeneous pivot trigger.** *(resolves Q1)* Stay organism-centric multiplex. Pivot to heterogeneous multilayer **only** when we concretely need to *rank non-organism endpoints* (pathways, metabolites, drugs) — an explicit decision, never drift.
- **B5 — Node set is the UNION, with a per-layer coverage mask.** *(user call; completes B3; informed by the D2 coverage audit)* The shared node set `V` is the **union of all Disbiome taxa** (dual-rank per B1), **not** the intersection of what the layers cover. Rationale and mechanics:
  - **Intersection is rejected.** Requiring every node to be present in every layer would keep only ~29% of taxa (the "all-layers" set from the audit), and — critically — those survivors are exactly the **well-studied taxa the D2 bias finding flags**. Intersection doesn't just lose ~70% of the disease taxa, it *concentrates the study-effort bias into the node set itself*, making every downstream claim "we recovered associations among the already-famous organisms." Rejected.
  - **Union is cheap because a node absent from a layer is simply isolated in that layer** (degree 0), never removed. This is the direct completion of **B3**: an isolated node means **unmeasured, never dissimilar**. Every layer therefore carries an explicit **coverage mask** (which nodes it actually measured), and **every analysis runs on that layer's measured-induced subgraph** — modularity sums per-layer so isolated nodes contribute nothing without distorting; MultiXrank walks route through the layers where a node *is* connected; **cross-layer congruence is computed on the pairwise intersection of just the two layers compared, never a global intersection.** So intersection happens locally, per comparison, exactly where it is meaningful.
  - **Rollup makes union nearly free at genus rank, but the relief is not free *signal*.** *(user point)* Because analysis is genus-primary (B1) with a species→genus aggregation rule, a **genus node lights up if *any* of its child species has data** — so effective genus coverage is materially higher than the raw species audit suggests, and taxa are "related to one another at some point in the hierarchy," letting information propagate along the NCBI/`TaxonClosure` tree. **This is why union costs us little.** The honest guard: **rollup *up* (species→genus aggregation) is legitimate; imputing *down* or *across* by relatedness is phylogenetic confounding** — precisely what the mandatory GTDB null layer ([`02`](02_retes_multiplex_model.md) §6) exists to control. Any edge whose support comes from hierarchical propagation must still be shown to **beat the phylogeny null** (A2), or it is just "related taxa look alike," not a finding. Propagation buys *coverage*, not *evidence*.
  - **Fully-dark nodes (~13%, only in the disease-target layer) are valid nodes but targets-only.** They can be asked "what explains this disease link?" and the honest answer — "nothing measured yet" — is itself an output: a **coverage-gap flag** marking a taxon that needs a genome / new data connector, not an uninteresting one. They cannot contribute to discovery and are labelled as such.
- **B6 — Signed and directed layers: keep the information, and match the method to the question.** *(resolves the open item flagged in [`04`](04_analysis_and_discovery.md))* Not every layer is a plain "these two are similar" edge. Some are **directed** — metabolic *exchange* runs one way (A feeds B is not the same as B feeds A). Some are **signed** — co-occurrence can be **co-presence (+)** or **co-exclusion (−)**, and the disease co-signature can be **concordant (+)** (both up, or both down, in a disease) or **discordant (−)**. The standard multiplex community-detection tool (MolTi) assumes edges are undirected, unsigned, and positive, so we must decide deliberately rather than quietly flattening. The rule:
  - **Always store the direction and sign** (the `directed` / `sign` fields, [`03`](03_layer_factory_and_catalog.md) §1a). Nothing is discarded at the source.
  - **Community detection (grouping organisms):**
    - unsigned, undirected layers (metabolic repertoire-sharing, phylogeny, ecophysiology) → standard multiplex modularity (MolTi), unchanged;
    - **signed** layers (co-occurrence, disease co-signature) → a **signed method**, where a **+** edge pulls two organisms into the same group and a **−** edge pushes them apart. Discarding the sign here would be wrong: co-exclusion is real anti-grouping information, not noise. **Concrete method (not a mystery tool):** split each signed layer into a positive-weight sub-layer and a negative-weight sub-layer and run `leidenalg.optimise_partition_multiplex` with `layer_weights=[+1, −1]` — i.e. optimize signed modularity Q⁺ − Q⁻ (Gómez–Jensen–Arenas), which is available in the same `leidenalg`/igraph stack already chosen ([`07`](07_stack.md) §6). No new external tool;
    - **directed** layers (metabolic exchange) → a **flow-based method that respects direction** (multilayer Infomap, already listed in [`04`](04_analysis_and_discovery.md)); or, when only coarse module structure is wanted, symmetrize — but only as an **explicit, stated** choice.
  - **Random walk / prioritization (MultiXrank):** the walk needs non-negative steps, so it travels along the **positive (and direction-respecting) edges**; a **co-exclusion (−) edge is not something you can "walk along"** and is handled as a **separate signal** (e.g. a penalty), never silently dropped.
  - **First-iteration pragmatism:** we may compute a quick baseline module structure by using magnitudes only, but we **must** also run the signed/directed-aware version and **report where sign or direction changes the result** — that disagreement is itself a finding (it echoes B1's "surface contradictions, don't average them away"), not something to hide.

## C. Layers

- **C1 — Abundance co-occurrence promoted to a CORE layer.** *(user call, Q9)* No longer "future." Rationale and specifics:
  - **The synergy that justifies early inclusion:** the **metabolic layer *predicts* interaction** (complementarity/exchange from reconstructions), while the **co-occurrence layer *observes* it** (organisms that actually co-vary across samples). **Congruence between predicted and observed interaction is a headline cross-layer analysis and a strong internal validation** — neither layer alone gives it.
  - **Sources:** prioritize **human-gut-relevant abundance** — `curatedMetagenomicData` and GMrepo abundance profiles — over environmental **Earth Microbiome Project** (EMP is breadth, not disease-relevant); **MicrobeAtlas** for scale.
  - **Inference method matters (correctness):** microbiome abundances are **compositional**, so naive Pearson/Spearman correlation is **forbidden** (spurious negative correlations). Use compositionally-aware inference — **SPIEC-EASI**, **SparCC**, or **FlashWeave**.
  - **Cost accepted:** this adds abundance-table ingestion to Mind and a network-inference step to the pipeline (Nextflow). Worth it for the observed-ecology signal and the metabolic-congruence check.
- **C2 — Updated core enrichment set** (see [`03`](03_layer_factory_and_catalog.md)):
  1. **`metabolic.repertoire`** (Jaccard over AGORA2 reaction sets) — function-sharing, feasible via `braidworks.fetch`, and the **Slice-2 gate layer**.
  2. **`metabolic.complementarity`** (SMETANA/MICOM community flux over paired AGORA2 SBML) — *predicted* interaction, the most mechanistic signal, but an **offline solver step, not a fetch**; second wave (Slice 4).
  3. **Abundance co-occurrence** (SPIEC-EASI/SparCC on curatedMetagenomicData/GMrepo) — *observed* ecology. **New core.** Its congruence with `metabolic.complementarity` is the headline cross-layer result — so that headline is iteration-1.5, since it needs both #2 and #3.
  4. **Phylogeny / GTDB** — the control/null layer; built on the new `gtdb.tree.rootpath` capability (per-entity tree placement, pairwise patristic distance reduced client-side — [`03`](03_layer_factory_and_catalog.md) §3.3).
  5. **Ecophysiology** (BacDive + ProTraits + Madin + FAPROTAX functional groups) — niche overlap.
  6. **BGCs** (antiSMASH/gutSMASH) — exploratory, second wave.
  Plus the **disease co-signature** target layer (from Mind).

## D. Data and interoperability

- **D1 — Disease vocabulary: MONDO (canonical) + MeSH (bridge); MedDRA-native first, MONDO later.** *(user call, Q10)* Standardize all sources onto **MONDO**, **keeping source-native codes as provenance**. Practical findings and the resulting sequence (see [`../data/disease_mapping/`](../data/disease_mapping/README.md)):
  - **MeSH is the bridge, not MedDRA.** Disbiome codes diseases in **MedDRA**, but MedDRA is **licensed**, so MONDO's free release carries only ~1,478 MedDRA cross-references — too sparse for a code join (and EBI's OxO bridge is currently offline). MONDO's **MeSH** coverage, by contrast, is dense (~24.8k, mostly exact). So the working bridges into MONDO are **name/synonym match + MeSH**; MedDRA is corroboration/provenance only. GMrepo is MeSH-coded natively, and the literature validation ([`05`](05_validation.md) §4) works in MeSH, so MeSH is needed as an interop layer regardless.
  - **The deterministic code join needs UMLS.** `MedDRA → UMLS CUI → MONDO` (UMLS is the hub both map into) would push coverage toward ~100% of coded diseases; it needs a free UMLS API key and is a **pending follow-up**, not a blocker.
  - **Interim crosswalk exists now:** an automated label/synonym+MeSH pass produced **222 confirmed** Disbiome→MONDO mappings (covering **77% of all experiments** — every high-volume disease), plus a flagged `review` (84) and `unmapped` (49) tail. Committed under `data/disease_mapping/`.
  - **Sequencing (why this isn't on the build's critical path):** iteration 1 is **Disbiome-only**, so diseases are keyed by their **native MedDRA code**; MONDO is a *normalization layer* that plugs in when a **second** disease source (GMrepo/gutMDisorder) arrives — which is the first thing that actually *needs* a shared vocabulary. Until then, the recovery split's "keep look-alike diseases together" guard (A4) uses a license-free **shared-organism-overlap fallback** instead of the MONDO hierarchy. See [`10`](10_architecture_and_roadmap.md).
- **D2 — Coverage audit is the first empirical task.** *(resolves Q8 as an action, not a decision)* Before trusting any layer: for each layer, measure **what fraction of Disbiome taxa (at genus and species) actually get enriched** — have genomes/reconstructions/traits/BGCs/abundance profiles — and **whether that coverage is biased** toward well-studied organisms. A layer that only covers the already-famous taxa is close to useless and we need to know that up front.
